# SP-0 + SP-1 — Foundation (Toolchain & Universal Substrate)

- **Date:** 2026-06-29
- **Status:** Approved (brainstorming) — ready for implementation plan
- **Parent:** `2026-06-29-excelents-modernization-architecture.md`

This spec covers the first buildable unit of the `excelents` rewrite: **SP-0**
(repo transform + toolchain + CI, producing an empty-but-releasable skeleton) and
**SP-1** (the universal, web-standard substrate everything else builds on).

---

## SP-0 — Toolchain & scaffold

### Objective

Transform this repo in-place into a modern TypeScript library skeleton that builds,
tests, lints, formats, type-checks, and *releases* — with no real features yet. Proves
the entire toolchain end-to-end before any OOXML code exists.

### Repo transform (in-place)

- **Add:** `src/`, `test/` (with `test/fixtures/`, `test/conformance/`),
  tool configs, CI.
- **Keep:** the 35 `.xlsx` + 1 `.csv` fixtures (moved to `test/fixtures/`), `LICENSE`,
  git history.
- **Delete:** `lib/`, `gruntfile.js`, `.babelrc`, `.browserslistrc`, `excel.js`,
  the old root `index.ts`, the hand-written `index.d.ts`, `benchmark.js`, the old
  `spec/` harness, `.eslintrc`/`.prettier*` legacy configs.
- **Oracle:** add `exceljs@4.4.0` as a **devDependency**.

### Tool configuration (intent — exact values finalized in the plan)

- **`package.json`:** `"type": "module"`, `"sideEffects": false`,
  `"engines": { "node": ">=20" }`, `"packageManager": "pnpm@11.x"`. The `exports`,
  `main`, `module`, `types` fields are **auto-generated by tsdown** (`exports:true`) —
  not hand-maintained. `files` limited to `dist` + `LICENSE` + `README`.
- **`tsdown.config.ts`:**
  ```ts
  import { defineConfig } from 'tsdown'
  export default defineConfig({
    entry: ['./src/index.ts', './src/node.ts', './src/csv.ts'],
    format: ['esm', 'cjs'],
    platform: 'neutral',
    dts: true,                // → oxc-transform via isolatedDeclarations
    exports: true,            // auto-write package.json exports/main/module/types
    clean: true,
    publint: 'ci-only',
    attw: 'ci-only',
    target: 'node20',
  })
  ```
- **`tsconfig.json`:** `strict: true`, `isolatedDeclarations: true`,
  `declaration: true`, `module: 'ESNext'`, `target: 'ES2022'`,
  `moduleResolution: 'bundler'`, `verbatimModuleSyntax: true`, `include: ['src']`
  (scoped to `src` to avoid the tsgolint "unmatched files" crash).
- **`vitest.config.ts`:** minimal; a separate browser project (`@vitest/browser`,
  Playwright provider) for the browser-targeted substrate tests.
- **Lint:** `oxlint` with `--type-aware` (via pinned `oxlint-tsgolint`).
- **Format:** `oxfmt` (pinned).
- **TypeScript:** TS7 native preview (`@typescript/native-preview`, pinned dated
  build) used for `tsc --noEmit`.

### CI (`.github/workflows/ci.yml`)

Node ≥ 22.18 host. Steps: install (pnpm) → `typecheck` → `lint` → `format --check`
→ `test` → `build` → `size` → `publint` + `attw` (via tsdown ci-only). Pinned tool
versions; matrix includes a downstream type-compat job resolving the shipped `.d.ts`
against TS 5.x and 6.x.

### Conformance harness (scaffold)

`test/conformance/` provides helpers used from SP-3 onward, scaffolded here:

- `withOracle(buildFn)` — builds a workbook with both old `exceljs` and new
  `excelents`.
- `unzipToParts(bytes)` — returns a normalized map of `{ partName → canonical XML }`.
- `diffParts(a, b)` — structural XML diff with normalization (attribute order,
  whitespace, namespace prefixes).
- `parseFixture(path)` — loads a `test/fixtures/*.xlsx` for read-path tests.

In SP-0 these are stubs with their own unit tests (verifying the diff/normalize logic
against synthetic XML), so the harness is trustworthy before real codecs arrive.

### SP-0 acceptance criteria

1. `pnpm build` emits ESM + CJS + `.d.ts` for all three entries; `dist` has **zero
   runtime dependencies** (verified by inspecting output imports).
2. `pnpm typecheck`, `pnpm lint`, `pnpm format --check`, `pnpm test` all pass on the
   stub.
3. `publint` + `attw` pass (correct dual-package exports/types).
4. `pnpm size` runs and reports per-entry gzipped sizes against a budget.
5. CI green, including the downstream TS-version compat job.
6. The package is `npm pack`-able and importable as both ESM and CJS in a smoke test.

## SP-1 — Universal substrate

### Objective

Implement the web-standard plumbing every higher layer needs: read/write the OPC
(xlsx) ZIP container, tokenize and emit XML in a streaming fashion, bridge to Web
Streams, abstract the filesystem behind an injected adapter, and provide core
spreadsheet utilities. **No object model, no OOXML semantics** — just the substrate.

All core code uses web-standard APIs only. Anything Node-specific lives in `src/node.ts`.

### Unit 1 — OPC container (`src/opc/`)

Reads and writes the Open Packaging Conventions ZIP used by xlsx.

- **ZIP container (owned):** local file headers, central directory, end-of-central-
  directory, **ZIP64** for large parts. Stored + DEFLATE entries.
- **Compression:** native `CompressionStream`/`DecompressionStream` (`deflate-raw`).
  An injectable `{ deflate, inflate }` codec interface defaults to native; `fflate` is
  the documented fallback (never a runtime dependency).
- **Package model:** `Package` with typed `Part`s; `[Content_Types].xml`
  (default + override content types); `_rels/*.rels` relationship graph; part-name
  resolution and URI handling.
- **API shape (illustrative):**
  ```ts
  interface OpcPackage {
    getPart(name: string): OpcPart | undefined
    addPart(name: string, contentType: string, data: BodyStream): OpcPart
    relationships(part?: OpcPart): RelationshipSet
    write(): ReadableStream<Uint8Array>
  }
  function readPackage(source: ReadableStream<Uint8Array> | Uint8Array): Promise<OpcPackage>
  ```
- **Streaming:** writing a package yields a `ReadableStream`; reading accepts a stream
  or bytes. Central-directory handling supports incremental writing for SP-7.

### Unit 2 — XML engine (`src/xml/`)

- **Streaming tokenizer (owned, ~300–500 LOC):** a pull/SAX tokenizer for the OOXML
  subset — elements, attributes, text, CDATA, the five predefined entities, namespaces.
  No DTD/processing-instruction/exotic-entity support (machine-generated xlsx XML never
  uses them). Emits tokens against a `ReadableStream`/string source.
- **Typed XML writer:** a string-builder-based writer with correct escaping, attribute
  ordering control, and namespace handling. SAX-style API so it can stream output.
- Both are designed to feed the streaming engine in SP-7 without buffering whole parts.

### Unit 3 — IO & adapters (`src/io/`)

- **Web Streams helpers:** chunking, concatenation, `Uint8Array` ↔ stream, base64
  encode/decode (for embedded media later), `TextEncoder`/`TextDecoder` wrappers.
- **Codec abstraction:** the pluggable `{ deflate, inflate }` interface (default
  native).
- **`FileSystemAdapter` interface:** `readFile`, `writeFile`, `createReadable`,
  `createWritable`. Core never imports `node:fs`; it consumes an injected adapter.
  `src/node.ts` (the `./node` entry) provides the Node implementation over `node:fs`
  and bridges `node:stream` ↔ Web Streams.

### Unit 4 — Utilities (`src/utils/`)

- **Address / col-cache:** `A1` ↔ `{ row, col }`, absolute/relative refs, range
  parsing/encoding (port of the proven `col-cache` logic, retyped).
- **Ranges:** range math used for dimensions/merges later.
- **Number-format helpers:** built-in format id table + format-code utilities (no
  rendering engine yet).
- **Serial-date math:** Excel serial ↔ `Date` for both 1900 and 1904 epochs (pure
  arithmetic; replaces `dayjs`).
- **Shared-strings table:** insertion + index lookup structure.

### SP-1 acceptance criteria

1. **Round-trip a ZIP:** write a multi-part package and read it back; parts, content
   types, and relationships are byte/structure-equivalent. ZIP64 path covered.
2. **Native + injected codec parity:** the same package round-trips with the native
   `CompressionStream` codec and with an injected (`fflate`) codec; outputs are
   inflate-equivalent.
3. **XML round-trip:** tokenize a representative OOXML fragment and re-emit it; the
   normalized result equals the input (validated with the SP-0 diff helper).
4. **Streaming:** large synthetic parts process without buffering the whole part in
   memory (bounded peak memory in a streaming test).
5. **Adapter isolation:** core (`src/index.ts`) imports **no** `node:` built-in
   (enforced by an oxlint rule / import check); Node behavior works only via
   `excelents/node`.
6. **Utils correctness:** address/range/serial-date/number-format helpers pass unit
   tests, including the 1900-leap-year bug compatibility and 1904 mode.
7. All SP-0 gates still green (types, lint, format, size, publint/attw).

### Boundaries & interfaces (for isolation)

Each unit is independently testable with a single clear responsibility:

| Unit | Does | Used via | Depends on |
|---|---|---|---|
| `opc/` | xlsx ZIP package read/write | `readPackage` / `OpcPackage` | `io/` codec + streams |
| `xml/` | streaming XML parse/emit | tokenizer + writer | `io/` streams |
| `io/` | Web Streams, codec, FS adapter iface | helpers + interfaces | platform only |
| `utils/` | address/range/date/format/sst | pure functions/classes | none |

Higher layers (SP-2+) depend only on these public interfaces, never on internals.

### Out of scope for SP-1

- Any object model (Workbook/Worksheet/Cell) — SP-2.
- Any OOXML semantics (worksheet/styles/workbook parts) — SP-3/SP-4.
- CSV — SP-6. Streaming workbook engine — SP-7.
